## 概要

- センターごとに商品群別の掛率を管理する
- 掛率は6ヶ月ごとに変更される可能性がある
- 過去の請求書を再発行する際、当時の掛率で計算できるよう履歴を保持する
- 請求書（表紙＋明細）を商品群別・税率別に出力する

---

## 1. DBテーブル作成

`sql/create_rates_table.sql` として新規作成。phpMyAdminで実行する。

```sql
CREATE TABLE center_rates (
  id INT AUTO_INCREMENT PRIMARY KEY,
  fc_user_id INT NOT NULL,
  rate_system DECIMAL(4,2) COMMENT 'システム商品掛率（例：0.50）',
  rate_general DECIMAL(4,2) COMMENT '一般商品掛率（例：0.60）',
  valid_from DATE NOT NULL COMMENT '適用開始日',
  valid_to DATE DEFAULT NULL COMMENT '適用終了日（NULLは現在有効）',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (fc_user_id) REFERENCES fc_users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 掛率の参考値

| センター種別 | システム掛率 | 一般掛率 |
|-------------|------------|---------|
| BC | 50%・55% のいずれか | 60%・65% のいずれか |
| FC | 65%・70%・75% のいずれか | 75%・80%・85% のいずれか |

※センターごとに個別設定。DBには 0.50 のような小数で登録する。

---

## 2. 商品群の定義

請求書の計算に使う商品群は4つ。`products` テーブルの `category` カラムで判定する。

| 商品群 | categoryの値 | 掛率 | 税率 |
|--------|------------|------|------|
| システム商品（化粧品） | 'システム' かつ tax_rate = 0.10 | rate_system を使用 | 10% |
| システム商品（健康食品） | 'システム' かつ tax_rate = 0.08 | rate_system を使用 | 8% |
| 一般商品 | '一般' | rate_general を使用 | 10% |
| 販促品・CS・備品 | 'CS', '販促', '備品' | 掛率なし | 10% |

### 販促品の価格選択ルール

- center_type = 'BC' → price_bc を使用
- center_type = 'FC' → price_fc を使用
- center_type = '販社' → price_hansha を使用

### 一般商品の+10%商品について

機械類・催事品など一部の一般商品は掛率に+10%が加算される。
対象商品はその都度異なるため、Excelに出力後に在りさんが手入力で修正する運用とする。
システム側での自動判定・フラグ管理は行わない。

## 3. APIファイル

### 3-1. `api/admin/rates.php`（新規作成）

#### GET 現在の掛率取得

```
GET api/admin/rates.php?fc_user_id=21
```

valid_to IS NULL のレコードを返す。

#### GET 履歴取得

```
GET api/admin/rates.php?fc_user_id=21&history=1
```

全レコードを valid_from DESC で返す。

#### POST 掛率登録・更新

```
POST api/admin/rates.php
Content-Type: application/json

{
  "fc_user_id": 21,
  "rate_system": 0.50,
  "rate_general": 0.60,
  "valid_from": "2026-06-01"
}
```

処理内容：
1. 現在有効なレコード（valid_to IS NULL）の valid_to に valid_from の前日をセット
2. 新レコードをINSERT（valid_to は NULL）

### 3-2. `api/admin/invoice.php`（既存を修正）

請求書発行時の掛率取得：請求対象月度の最終日を基準日として center_rates を検索する。

```sql
SELECT * FROM center_rates
WHERE fc_user_id = ?
AND valid_from <= '基準日'
AND (valid_to IS NULL OR valid_to >= '基準日')
ORDER BY valid_from DESC
LIMIT 1
```

掛率が未登録の場合はエラーメッセージを返す（請求書を発行しない）。

#### 表紙の金額計算ロジック

| 項目 | 計算方法 |
|------|---------|
| システム上代 | システム商品の単価×個数の合計（8%・10%合算） |
| システム下代 | 上代 × rate_system |
| 一般上代 | 一般商品の単価×個数の合計 |
| 一般下代 | 上代 × rate_general |
| 別口（販促）上代 | BC/FC価格 × 個数の合計 |
| 別口（販促）下代 | 上代と同額（掛率なし） |
| 当月御買上額 | システム下代＋一般下代＋別口下代 |
| 当月消費税額 | 8%税額＋10%税額 |
| 当月御請求金額 | 当月御買上額＋当月消費税額＋繰越額 |
| 8%対象下代 | システム健康食品の下代合計 |
| 8%税額 | 8%対象下代 × 0.08 |
| 10%対象下代 | システム化粧品下代＋一般下代＋別口下代 |
| 10%税額 | 10%対象下代 × 0.10 |

※前月御請求額・当月御入金額・繰越額・値引その他は手入力項目とする（初期値0）。

### 4-2. 請求明細（2ページ目以降）

#### ヘッダー部

```
請 求 明 細 書
[年] 年 [月] 月度（[年] 年 [月] 月 [日] 日発行）    8% 一販

お得意先名
  [センター名]
  [担当者名] 様            システム掛率 [0.50]   一般掛率 [0.60]
```

#### 明細行

| 注文日 | 商品コード | 商品名 | 単価 | 個数 | 区分 | 金額（下代） | 上代システム | 上代一般 | 上代販促 |
|--------|-----------|--------|------|------|------|------------|------------|---------|---------|

- 注文日：YYYY/MM/DD 形式
- 区分：システム／一般／別口 のいずれか
- 金額（下代）：単価×個数×掛率（販促は単価×個数のまま）
- 上代システム・上代一般・上代販促：該当する列のみ金額を入れ、他は空欄

#### フッター部（明細の末尾）

```
総合計      [下代合計]

合計 8%     [8%対象下代計]

合計 10%    [10%対象下代計]    [10%上代計（システム+一般+販促）]
```

- 一般商品掛率（数値入力、%表示。例：60）
- 適用開始日（日付入力）
- 「保存する」ボタン

※入力は%表示で受け取り、DB保存時に0.50に変換する。

#### 履歴一覧（変更フォームの下に表示）

表示項目：適用開始日・適用終了日・システム掛率・一般掛率
新しい順（valid_from DESC）で表示。
