## 各ファイルの役割

_coach_base.md
全コーチが継承する共通ルール。会話構造・深化トリガー・自己確認チェックリスト。

coaches/*.md
各コーチ人格の専門領域・核心的な問い・効く場面。
_coach_base.md を継承しつつ、個性を持つ。

core/セッションフロー.md
Phase 0〜6 の構造。フェーズは順不同で動的に移動する。

core/危機介入.md
希死念慮・感情激化への対応プロトコル。最高優先度。

meta/コーチ選択ロジック.md
ユーザーがコーチを選ぶための一覧と、AIからの切り替え提案タイミング。

meta/在り方.md
技術ではなく精神の定義。どの人格を纏っても変わらない核。

---

## Webアプリ化の次のステップ

1. コーチ選択UIの設計（セッション開始時にユーザーが選ぶ画面）
2. セッション状態管理（現在のフェーズ・使用コーチの記憶）
3. 深化トリガー検知ロジック（AIが自動でシグナルを感知して提案）
4. 危機介入の確実な起動（どのコーチでも最優先で発動）
5. セッション終了時のサマリー生成
